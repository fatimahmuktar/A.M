NEU AttendAI

Smart Geo-Fencing Attendance System

 

 

1. Executive Summary and Project Vision

NEU AttendAI revolutionizes classroom attendance management. It is not merely a tool for recording absence, but an integrated platform that uses modern technology to solve manipulation issues and ensure the integrity of the educational process. The core vision is to eliminate proxy attendance and human error by enforcing a double-verification protocol that combines Geo-Fencing (to confirm physical presence in the classroom) with a Dynamic QR Code validation.

 

The system is designed not only to record attendance but to act as a policy enforcement engine, providing faculty with a clear, real-time overview and maintaining their authority through a structured warning and deprivation system. This report details the architectural design, core technologies, and a phased implementation roadmap for a successful project delivery.

 

Key Features of the Proposed System:

• Zero Proxy Attendance: Achieved via mandatory GPS verification within a 50m classroom perimeter and a Dynamic QR Code that changes every few seconds.
• Advanced Security: Includes mechanisms for GPS Spoofing Detection and Impossible Travel Detection.
• SSO Integration: Seamless authentication for faculty using the university's Single Sign-On (SSO) system.
• Policy Engine: Automated enforcement of the 70% attendance threshold with a 3-stage warning system.
• Real-time Monitoring: A live dashboard showing attendance status and location data.

2.2 Core Logic: Geo-Fencing and AI-Powered Enforcement Engine

The system's reliability is built upon critical, advanced algorithms and Artificial Intelligence (AI) logic:

 

A. Haversine Geo-Fencing Algorithm

This proprietary algorithm is used to calculate the great-circle distance between two points on a sphere (the Earth).

 

• Function: Verifies that the student's current GPS coordinates are within a 50-meter radius of the registered classroom coordinates.
• Indoor Accuracy: Addresses weak indoor GPS signals by leveraging the Fused Location Provider which combines GPS, Wi-Fi, and cellular data for better accuracy.
 

B. Policy and AI-Powered Enforcement Engine

This engine applies the university's academic policy through statistical modeling and structured logic, incorporating advanced fraud detection:

 

• Anomaly Detection (AI Touch): Uses Impossible Travel Detection to compare the time and location of the current check-in with the student's last recorded location. If the distance is too great for the elapsed time (e.g., traveling between two buildings in one second), the system flags the attempt as fraudulent.
• GPS Spoofing Detection: The system includes a check for common indicators of location spoofing (e.g., detecting if "Developer Options" are enabled on Android devices), adding an extra layer of security.
• 70% Threshold: Automatically tracks each student's attendance rate against the required 70% minimum.
• 3-Warning System: Implements a progressive warning system (e.g., Warning 1 at 80%, Warning 2 at 75%, Warning 3 at 70%).

3.1 Data Security and Privacy

The system is designed with a strong commitment to student privacy. Location tracking is strictly limited to the moment of attendance registration. No continuous tracking of student location is performed outside of the check-in process. All data is anonymized and stored securely, complying with standard university data protection policies.
